üéØ MISSION 5.1: Form Structure & Labels

üìñ THE KNOWLEDGE: Form Foundation

Forms collect user input and send it somewhere (usually a server).

üèóÔ∏è Essential Elements:

  • <form> - Container for all form elements
  • <label> - Describes what an input is for
  • <input> - Various input fields
  • <button> - Submit button

üìù Basic Form:

<form>
  <label for="username">
    Username:
  </label>
  <input type="text" 
         id="username" 
         name="username">
  
  <button type="submit">
    Submit
  </button>
</form>

üé® Live Demo:

üí° Why Labels Matter:

  • for attribute must match input's id
  • Clicking label focuses the input (better UX!)
  • Screen readers read labels to users
  • Required for accessibility

üîë Input Attributes Explained:

  • type - What kind of input (text, email, password, etc.)
  • id - Unique identifier (links to label)
  • name - Field name sent to server (CRITICAL!)
  • placeholder - Hint text inside input

üéØ MISSION 5.4: Form Validation & Attributes


‚úÖ Built-in Validation

HTML5 can validate forms WITHOUT JavaScript!

Attribute What It Does Example
required Must be filled out <input required>
minlength Minimum characters minlength="8"
maxlength Maximum characters maxlength="20"
min / max For numbers min="1" max="100"
pattern Regex validation pattern="[0-9]{3}"
placeholder Hint text placeholder="Enter..."

‚ú® Complete Example:

<form>
  <label for="username">Username (4-12 chars):</label>
  <input type="text" 
         id="username" 
         name="username" 
         minlength="4" 
         maxlength="12" 
         required 
         placeholder="johndoe">
  
  <label for="age">Age (18+):</label>
  <input type="number" 
         id="age" 
         name="age" 
         min="18" 
         max="120" 
         required>
  
  <label for="email">Email:</label>
  <input type="email" 
         id="email" 
         name="email" 
         required>
  
  <button type="submit">Register</button>
</form>

üé® Other Useful Attributes

  • disabled - Makes input uneditable and grayed out
  • readonly - Can view but not edit
  • autofocus - Automatically focuses this field on page load
  • autocomplete="off" - Disables browser autofill
  • value - Pre-fills the input with a default value
